We're excited to introduce support for handling log sources containing timestamps from multiple time zones using custom OpenTelemetry Collector YAML configurations. This enhancement enables you to parse logs from different regions within a single source file, beyond the default single time zone configuration available in Source Template settings. [Learn more](/docs/send-data/opentelemetry-collector/remote-management/source-templates/otrm-time-reference/#parsing-multiple-time-zones-from-a-single-source).
